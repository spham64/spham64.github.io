S&P 500 ESG Clustering

Introduction

ESG stands for Environmental, Social, and Governance, which is a set of criteria that is used by investors to evaluate a companies performance and sustainability beyond financial metrics.

  • Environmental: focus on company's impact on the planet
    • Climate risk
    • Energy use
    • Pollution
    • Waste
    • Resources
  • Social: how a company manages its relationships with employees and the community
    • Employee relations
    • Community relations
    • Human rights
    • Customer satisfaction
  • Governance: examines how a company is run
    • Leadership
    • Executive pay
    • Audits & controls
    • Shareholder rights
    • Transparency

This project will explore ESG performance of the S&P 500 companies through K-Means clustering, aiming to identify which sectors and industries fall into specific ESG risk clusters. By implementing and analyzing a clustering model, we can help investors see which areas of the market have historically high or low ESG and base their investments on those factors.

What is Clustering?

Clustering is an unsupervised machine learning modeling technique that groups observations based on similarities. The goal of this method is to find natural groupings within a dataset without any prior knowledge / labels of the categories. In this project, I will be using K-Means clustering.

K-Means Clustering: create K amount of random center points (centroids). In the algorithm, data points are compared to the centroids. The algorithm tries to optimize placement for each centroid.

The Data

It provides key features such as Symbol, Name, Industry, Environmental Risk Score, Social Risk Score, Governance Risk Score, etc. In total, there are 15 key features in this dataset with over 500 rows.

K-Means Model

Now that we have cleaned the data and explored some correlations using our data visualizations, we can start to build our model. For this experiment, I will use a K-Means clustering model.

Selecting Features for Clustering

In[47]:
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.decomposition import PCA

features = [
    'Environment Risk Score',
    'Social Risk Score',
    'Governance Risk Score',
    'Controversy Score',
]

X = df[features]

Since Total ESG Risk Score is strongly correlated with others, we can omit it to avoid redundancy.

Elbow Method

In[48]:
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

inertia = []
k_values = range(1, 10)

for k in k_values:
    kmeans = KMeans(n_clusters=k, random_state=42)
    kmeans.fit(X_scaled)
    inertia.append(kmeans.inertia_)

plt.plot(k_values, inertia, marker='o')
plt.xlabel('Number of Clusters (k)')
plt.ylabel('Inertia')
plt.title('Elbow Method for Optimal k')
plt.show()

Since K-Means is distance based, the features are scaled the same. The next block of code is the Elbow Method which helps decide how many ESG clusters to form. Looking at the graph, we can see the "elbow" point is at k=3 or k=4. For this project lets use k=3.

Applying K-Means Clustering

In[51]:
kmeans = KMeans(n_clusters=3, random_state=42)

Impact

This project aims to build a K-Means clustering model that groups S&P 500 comapnies / sectors based on their Environmental, Social, and Governance (ESG) performance. Understanding how these sectors and companies cluster according to ESG risk helps investors investors, policymakers, and analysts identify patterns across industries and make more informed decisions. However, there are potential social and ethical considerations that arise from this project. Relying solely on algorithmic clustering may oversimplify complex ESG behaviors or misclassify companies due to incomplete or outdated data. Therefore, these results should be used as a guiding analytical tool as the goal of this project is primarily educational and exploratory.

Conclusion

By experimenting and understanding ESG features and clustering parameters, I learned that the choice of features and the number of clusters greatly influences how companies are grouped and interpreted. Overall, this project showed me the importance of exploring relationships and patterns rather than making exact predictions.
